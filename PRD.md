**Author**: Claude (PM) + Gemini (Market Researcher)
**Status**: Draft

---

## 1. 背景・問題定義

### 課題

AI時代において、エンジニアがMarkdownファイルを扱う機会は急増している。
Claude・ChatGPTなどのAIが生成したドキュメント、README、技術仕様書を
**快適に閲覧・編集できるMacネイティブツールが存在しない**。

### 既存ツールの問題

| ツール | 問題 |
|--------|------|
| Typora | Electron製で重い。拡張性なし |
| Bear | 独自DB管理でGit連携不可。フォルダ管理なし |
| Obsidian | 多機能すぎて複雑。学習コスト高 |
| iA Writer | シンプルすぎる。AI連携なし。エンジニア向け機能不足 |
| VS Code | Markdown専用ではない。オーバースペック |

### 市場の空白

「Macネイティブの軽快さ」×「シンプルなUI」×「フォルダベース管理」×「AI連携」×「Markdownネイティブdiffビューア」
を同時に満たすエディタは存在しない。

特に**Markdownレンダリング済みdiff**（rawではなくプレビュー表示で差分を見る）は
GitHub・VS Code・Typora含め現状どのツールも提供していない。

---

## 2. プロダクトビジョン

> **AIが生成したMarkdownを、Macで最も快適に扱えるエディタ**

エンジニアが技術ドキュメント・AI生成コンテンツを
ストレスなく閲覧・編集・管理できる、Macネイティブの軽量エディタ。

---

## 3. ターゲットユーザー

### Primary
- **自分自身**（開発者・作者）
- AIツールを日常的に使い、生成されたMdファイルを頻繁に扱うエンジニア

### Secondary
- 技術ドキュメントを書くエンジニア全般
- OSSコントリビューター（README・CHANGELOG管理）

### ユーザーが感じている不満（Geminiリサーチより）
- Electronアプリの起動が遅い
- フォルダ管理とGit連携が同時にできない
- シームレスプレビューがない or 編集と分離されている
- コードブロックのシンタックスハイライトが不十分
- AIを使いたいが外部ツールへの切り替えが煩雑

---

## 4. 差別化ポジション

```
              シンプル
                 ↑
         [kobaamd]  iA Writer
                 |
フォルダ管理弱 ←─────────→ フォルダ管理強
                 |
    Bear    Obsidian
                 ↓
              高機能・複雑
```

**ポジション**: シンプル × フォルダ管理強 × AI連携 × Macネイティブ

## 6. 機能要件

### v1.0（MVP）

- [ ] Markdownリアルタイムプレビュー（2ペイン or シームレス選択可）
- [ ] シンタックスハイライト（コードブロック・言語別）
- [ ] 基本的なMarkdown記法のサポート（CommonMark準拠）
- [ ] Mermaid.jsダイアグラムのレンダリング

#### ファイル管理
- [ ] フォルダツリー表示（左サイドバー）
- [ ] フォルダを開いてファイル一覧表示
- [ ] ファイル作成・削除・リネーム
- [ ] 全文検索（フォルダ内）

#### Mac最適化
- [ ] SwiftUIネイティブ実装
- [ ] macOSキーボードショートカット完全対応
- [ ] ダークモード / ライトモード対応
- [ ] メニューバー統合

#### AI連携（v1から）
- [ ] 選択テキストをAIに投げて補完・リライト
- [ ] ドキュメント要約
- [ ] AIプロバイダー設定（OpenAI / Anthropic / Gemini 切り替え可）

### v2.0（拡張）
- [ ] **Diffビューア**（差別化機能・優先度高）
  - **Git diff**: コミット間の差分をエディタ内で視覚表示（git diffの代替）
  - **AI前後diff**: AIリライト前後で何が変わったか即座に確認
  - **ファイル間diff**: 2つのMdファイルを並べて差分表示
  - **PRレビューdiff**: GitHubのdiffより読みやすいMarkdown特化の差分表示
  - 表示形式: サイドバイサイド（左: before / 右: after）
  - 追加行（緑）・削除行（赤）のハイライト
  - **Markdownレンダリング済みdiff**（rawテキストではなくプレビュー状態で差分表示）
- [ ] Git統合（コミット・ステージング）
- [ ] 画像の自動保存（./assets への貼り付け）
- [ ] カスタムテーマ
- [ ] Vimキーバインドモード
- [ ] プラグインAPI

## 7. 非機能要件

- **プラットフォーム**: macOS 14 (Sonoma) 以降
- **言語**: Swift / SwiftUI
- **配布**: OSS（MIT License）、GitHub公開
- **データ管理**: ローカルファイルのみ（独自DBなし）
- **依存関係**: 最小限（隠しフォルダ・設定ファイルを作らない）

---

## 8. 非ゴール（やらないこと）

- Windows / Linux対応
- クラウド同期機能（iCloud連携は検討）
- リアルタイム共同編集
- 独自ファイルフォーマット（プレーンな.mdのみ）
- モバイル対応

## 9. 技術スタック（案）

| レイヤー | 技術 | 理由 |
|---------|------|------|
| UI | SwiftUI | Macネイティブ・高速・HIG準拠 |
| Markdownパーサー | cmark-gfm or swift-markdown | 高速・GFM対応 |
| シンタックスハイライト | Highlight.js or TreeSitter | 多言語対応 |
| ダイアグラム | Mermaid.js（WKWebView経由） | 実績あり |
| AI連携 | REST API（OpenAI/Anthropic/Gemini） | プロバイダー非依存 |

※ Architect（Claude）が技術選定を最終決定する
