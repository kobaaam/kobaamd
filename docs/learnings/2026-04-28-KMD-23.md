## 改善点

- **DragGesture の累積値 vs 差分値の誤用はSwiftUI の典型的な罠**: `value.translation` が累積値であることを実装プロンプトに明記していれば、初回で正しい実装が出ていた可能性が高い。Codex CLI へのプロンプトに SwiftUI DragGesture の注意事項が不足していた。
- **PRD の AC に書かれた要件（アウトライン空時の自動縮小）が初回実装で漏れた**: AC は6項目あり、うち1項目がレビューまで未実装だった。実装エージェントが AC を1つずつチェックリストとして消化するプロセスが不足していた。
- **テストが初回コミットに含まれていなかった**: ロジック変更を伴う PR にはテストを同時に含めるべきだが、レビュー指摘されるまで追加されなかった。

## 教訓

- **SwiftUI の DragGesture では `value.translation` は累積変位であり、毎フレームの差分ではない**: `onChanged` で状態に直接適用すると振動バグになる。開始時の値を保存して計算する必要がある。この知見は他の DragGesture 実装（スプリットビューのリサイザーなど）にも適用できる。
- **PRD の AC は実装チェックリストとして活用すべき**: 実装完了時に AC を1項目ずつ確認するステップを入れることで、レビュー前に未達を検出できる。
- **影響範囲マップの「変更してはいけない箇所」は非常に有効**: 大規模リファクタリングでもコラテラルダメージをゼロに抑えられた。今後の PRD でも必ず記載すべき。
